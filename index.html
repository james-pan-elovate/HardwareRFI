Benchmark Test Planning

Physical test scenarios grouped by deployment context. Each test bundles the capabilities that can be validated in one setup — so you know how many site installations you actually need.

⚑
Group capabilities into physical test scenarios. Each card below is one deployable setup — the signalized-intersection card, for example, validates red-light, red-light-with-speed, SSV, block-the-box, NTOR, crosswalk, and VLPD phase detection all from the same camera. Use this tab to plan how many distinct site setups you need, who leads each one, and which vendors are participating.
Capability labels link back to rows on tabs 01–06. MUST/NICE/ROAD weighting is inherited from the Priorities tab. Your planning edits are saved in this browser's local storage (same mechanism as Priorities) — use Save as Shareable HTML on the Priorities tab to bake a snapshot into a file colleagues can open.

🚦 Signalized Intersection (Full Suite) Planning
Fixed-pole camera at a signalized (traffic-light) intersection. One physical deployment validates the entire red-light family: front/rear red-light capture, red-light-with-speed, in-camera secondary speed validation at the stop-bar, block-the-box, NTOR/illegal turn, crosswalk encroachment, plus signal-phase detection (VLPD) and A/B shot timing. Wrong-way and restricted-lane monitoring can also be observed if the site geometry permits.

ALPR can be co-validated at the same setup since midblock geometry is ideal for plate capture.
